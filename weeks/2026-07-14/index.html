Week 1

Agent concepts and an observable ADK run

July 14, 2026 · Search Agent Study Group

Materials

  • Week 1 resources The living Notion view for selected references, examples, and demo material.

Goals

  1. See the broader landscape of components that may appear in an agentic system and in later study-group sessions.
  2. Establish shared definitions for the foundational concepts behind agents and tool use.
  3. Connect those concepts to one observable Google ADK run, from definitions in code to runtime events.

Agentic system landscape

A five-minute roadmap, not a checklist. An agent may use only a small subset of these components; later sessions can revisit each area in depth.

Session plan

  1. 5 min

    Map the agentic-system landscape

    A quick tour of the possible study-group syllabus: capabilities, runtime, state, control, and evaluation.

  2. 30 min

    Build the foundational vocabulary

    Introduce the LLM, agent, tool, skill, and ReAct concepts that the rest of the session will use.

  3. 15 min

    Observe a Google ADK agent run

    Connect agent, tools, app, session, runner, and run definitions to live events in ADK Web—including the visible instruction context, tool declarations and calls, results, and session flow.

The planned content uses about 50 minutes, leaving roughly 10 minutes for transitions, questions, and the optional Week 1 assignment. The demo will expose observable runtime data, not private model reasoning.

Concepts introduced

  • Augmented LLM

Assignment

Set up Google ADK and complete your first live agent checkpoint

This is an optional, lightweight Week 1 assignment. The goal is to confirm that your local learner environment can run a real ADK agent, observe its tool call safely, and receive your own agent codename.

Complete the assignment

  1. Follow the Google ADK Setup Playbook to clone the learner repository and create its isolated environment.
  2. Create and store your own Gemini API key locally.
  3. Run notebooks/00_setup_check.ipynb through the live checkpoint.

You are done when

The notebook shows the redacted agent timeline, confirms the expected tool call and result, and generates your personalized Week 1 agent codename. Your codename will be different from the maintainer example.

Optional public check-in

The notebook also generates a prefilled GitHub Issue Form. Submitting it is optional and separate from completing the setup. If you submit it, GitHub Actions validates the checkpoint, celebrates your codename, and closes the issue automatically.
