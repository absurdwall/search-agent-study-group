Week 1

Agentic AI systems and a complete ADK run

July 14, 2026 · Search Agent Study Group

Materials

  • Google ADK shop-agent notebook The instructor demo used for the prompt-to-output walkthrough.
  • Search Agent Lab The clean learner repository for setup checks and later assignments.
  • Google ADK: Get started with Python Public reference for setting up and running an ADK agent.
  • Search Agent Group ADK playbook Our living, tested setup guide for the study group.

Goals

  1. Build a high-level map of an agentic AI system and why each component matters. This will serve as a roadmap for later weeks.
  2. Walk through one complete agent run in Google ADK, from the user prompt to the final response.

Coverage plan

  1. Map the core system components: model and instructions, tools, orchestration, state and memory, approvals and guardrails, and evaluation.
  2. Use the shop-agent notebook to follow an initial user prompt, model requests, tool calls, agent delegation, and the final response.
  3. Connect the walkthrough back to the system map and identify questions for future weeks.

Assignment

Set up Google ADK and complete your first live agent checkpoint

This is an optional, lightweight Week 1 assignment. The goal is to confirm that your local learner environment can run a real ADK agent, observe its tool call safely, and receive your own agent codename.

Complete the assignment

  1. Follow the Google ADK Setup Playbook to clone the learner repository and create its isolated environment.
  2. Create and store your own Gemini API key locally.
  3. Run notebooks/00_setup_check.ipynb through the live checkpoint.

You are done when

The notebook shows the redacted agent timeline, confirms the expected tool call and result, and generates your personalized Week 1 agent codename. Your codename will be different from the maintainer example.

Optional public check-in

The notebook also generates a prefilled GitHub Issue Form. Submitting it is optional and separate from completing the setup. If you submit it, GitHub Actions validates the checkpoint, celebrates your codename, and closes the issue automatically.
