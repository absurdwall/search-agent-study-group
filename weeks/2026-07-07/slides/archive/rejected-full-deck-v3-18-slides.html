Decision before design

Should this even be an agent?

Autonomy is not free. It buys flexibility by spending latency, cost, and control.

Start with the simplest system that can do the job. Add agency only when the task demands dynamic decisions.

Decision engine

Model choice matters, but instructions define the job

The model is the decision engine. The task contract, instructions, and policies tell it what decisions are allowed to mean.

Fast model

High-volume chat, low-risk routing, simple extraction.

Reasoning model

Planning, multi-tool use, code, high-cost mistakes.

Escalation

Use humans when uncertainty or impact crosses a boundary.

Context, state, memory

The agent can only decide from the state it can see

Context engineering is the work of choosing what enters the model's limited attention at each step.

State and memory are not one feature. They are a strategy for preserving the right information while the loop keeps moving.

Task state

Current goal, constraints, unresolved decisions, last observation.

+
Retrieved context

Files, policies, search results, database records, examples.

+
Persistent memory

Notes, user preferences, project state, summaries, checkpoints.

=
Decision context

The actual information available for the next model call.

Takeaway: more context is not always better; high-signal context is better.

Evaluation mental model

How do we know the agent actually works?

01

Outcome quality

Did the final state satisfy the task, not just sound plausible?

02

Trajectory quality

Did it choose the right tools, valid arguments, routes, and recovery steps?

03

Safety and control

Did it respect policies, permissions, approvals, and action boundaries?

04

Efficiency

How many steps, tokens, tool calls, seconds, and dollars did it use?

Takeaway: agent evals inspect the result and the path.

Production quality is a tradeoff surface

Reliability

Reliability

Better reasoning, more checks, approvals, and retries can improve quality.

Latency

Every extra model call, tool call, and human gate adds time.

Cost

Longer traces, larger context, and stronger models must justify their value.

Design task

Choose the acceptable failure boundary for this product and user.

Takeaway: production agent design is optimization under constraints.

Reusable checklist

Questions to ask about any agent

1What task and outcome is the system responsible for?
2Does this need an agent, or would a simpler workflow work?
3What model and instructions make decisions?
4What context, state, and memory are available at each step?
5What tools and actions can the system access?
6Who controls the loop, handoffs, and stopping?
7What permissions, guardrails, and approvals apply?
8How will we trace, evaluate, and improve it?
9What are the acceptable cost, latency, and failure boundaries?
End frame: inspect the system, not the demo sparkle.
