# Weekly Study To HTML Slides

Use this workflow when preparing a Search Agent Study Group session from user-selected material.

## Workflow

1. Confirm the provided material and source boundaries.
2. Read the material and produce a first-pass concept map before asking for optional input.
3. Walk through the material with the user, resolving concepts and noting open questions.
4. Add targeted extended reading only for concepts that need support.
5. Create or update the dated weekly folder under `weeks/YYYY-MM-DD-topic-slug/`.
6. Draft `slide-brief.md` before building slides.
7. Build the deck under `weeks/YYYY-MM-DD-topic-slug/slides/`.
8. Verify the deck in a browser before final delivery.
9. Keep the deck URL stable once it has been published or attached to Notion.

## Study Output

Capture:

- thesis and context
- key terms
- mechanism or architecture
- examples
- what is new or non-obvious
- limitations and critiques
- useful analogies
- presentation-worthy moments

## Slide Output

Prefer:

- progressive disclosure
- animated system diagrams
- live or simulated demos
- code or trace panes when useful
- discussion prompts
- speaker notes outside visible slide copy
- a stable `index.html` entry point for GitHub Pages
- direct HTML/CSS/JS rather than PPTX tooling by default

Avoid:

- dumping the source outline into slides
- overbuilding visual effects that do not clarify the point
- adding more sources just because they exist
